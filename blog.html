Something about let,var & const

Var

Function-Scoped: Variables declared with 'var' are function-scoped, which means they are only accessible within the function in which they are defined.

Hoisting: Variables declared with 'var' are hoisted to the top of their containing function or global scope. This means you can access a var variable before it's declared, but it will have an initial value of 'undefined' until it's assigned a value.

Reassignment: You can reassign 'var' variables within their scope. until it's assigned a value.

Let

Block-Scoped: Variables declared with 'let' are block-scoped, which means they are only accessible within the block (e.g., if statement, for loop, or function) in which they are defined.

Hoisting: Like 'var', 'let' variables are hoisted, but they are not initialized until the line of code where they are declared is executed. This is known as the "temporal dead zone."

Reassignment: You can reassign 'let' variables within their block scope.

Const

Block-Scoped: Variables declared with 'const' are also block-scoped, just like 'let'

Immutability: 'const' variables are meant to be constants, meaning their value cannot be reassigned after declaration. However, for objects and arrays declared with 'const', you can still modify their properties or elements.

Reassignment: Similar to 'let', const variables are hoisted but not initialized until the line where they are declared is executed.

Something About Null and Undefined

Null

Intentional Absence of Value: 'null' is often used to indicate that a variable intentionally lacks a value or that a value is explicitly empty.

Initialization: You can initialize a variable with 'null' when you want to signify that it will be assigned a meaningful value later.

As a Placeholder: Sometimes, 'null' is used as a placeholder for an object or value that is expected but not available at the moment.

Comparisons: You can compare a variable to 'null' to check if it has been assigned a meaningful value yet.

Undefined

Implicit Absence of Value: 'undefined' typically indicates that a variable has been declared but has not been assigned any value. It represents the implicit absence of a value.

Function Parameters: Function parameters that are not provided when the function is called are 'undefined' by default.

Array Elements: If you access an index in an array that does not exist, you'll get 'undefined'.

Object Properties: When you try to access an object property that doesn't exist, it returns 'undefined'.

Something About REST API

REST API stands for Representational State Transfer Application Programming Interface. It is a set of architectural principles and constraints for designing networked applications and services. REST is not a specific technology or protocol; rather, it is a style of designing web services that rely on the existing protocols and standards of the World Wide Web, primarily using HTTP (Hypertext Transfer Protocol).
